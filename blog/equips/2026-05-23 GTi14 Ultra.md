最近在走公司注册流程。

忍不住夸一嘴，杭州的政务系统做得实在是漂亮：几乎所有流程都支持线上办理，社交媒体上也有很多人分享详细的办理步骤，跟着网上的帖子一路就能顺下来，几乎没遇到过什么卡点。

直到前天去招行开完公户，才突然一拍脑袋，意识到财务管理的平台只有在 Windows 端才能正常运行。而我手头只有一台服役了五年的 MacBook Air M1。

## 我到底需不需要一台 Windows 主机？

一直以来，我的 Windows 之路就走得并不十分顺利：

![无数次写好装机配置清单，无数次地在理想和现实间权衡抉择](/content-assets/2026-06-01.png)

这几年，我的主力设备一直在 Mac 生态里，Coding、写作、项目管理、网站开发、内容创作，我目前的大部分工作实际上都可以在 Mac 上完成。为了弥补 M1 芯片性能的过时，我在最近还购置了一台新的 Mac mini，作为长期主力工作站。

因此，最开始的问题并不是“我要不要一台高性能 Windows 电脑”，而是：

**我到底有哪些场景一定需要 Windows？**

我一开始想到的场景包括：

1. 政务、税务、银行、公司办事系统。
2. 一些 Windows 专属软件。
3. 轻度 CAD。
4. 浏览器自动化和插件测试。
5. Steam、Ubisoft 里的部分游戏（这真的很重要）。
6. 未来可能需要 CUDA 或本地 AI 工具。
7. 作为一台可以长期挂着的 Windows 节点。
8. 各种各样意想不到的需求——

![好笑的是，没有 Win 我连这个 Vibe Coding 界的时尚单品都没法配置驱动](/content-assets/miniKB.jpg)

在这个阶段，我还没有完全明确这台机器的定位，所以自然而然会往“全能主机”的方向预估。

## 从传统台式机开始

最初的方案是自己装一台 Windows 主机。

和往常一样，我又写了一份理想当中的装机配置单。当时大概考虑过：

1. Ryzen 7 9700X。
2. B850M 主板。
3. 32GB 或 64GB DDR5 内存。
4. 1TB 或 2TB SSD。
5. RTX 5060 Ti 16GB，也考虑 RTX 5070 和 RTX 5070 Ti。
6. 750W 或 850W 金牌电源。
7. 联力 LANCOOL 216 机箱。

这套方案的优点很明确：性能强，扩展性好，散热好，稳定，未来想升级也方便。

但问题也很明显：

如果我之后还要买 Mac mini，那这台 Windows 主机一旦堆到这个配置，总预算就会非常难看，而且获得的成效除了能运行更顺畅的 3A 大作外并不明显。更重要的是，我的主力 Coding 已经在 Mac 生态里，Windows 这台机器如果配成高性能主力机，就会和未来的 Mac mini 产生生产力上的大量重叠。

## 重新定义这台新机器的角色

继续梳理后，我发现自己对 Windows 的需求其实可以分层。

第一层是刚需：

1. 政务、税务、银行控件。
2. 企业办事系统。
3. Windows 软件兼容。
4. 浏览器插件和自动化测试。
5. 轻度 CAD。

第二层是可选：

1. 策略游戏。
2. 荣耀战魂、彩虹六号这类老网游。
3. 后端测试环境。
4. Docker、Playwright、定时任务。
5. 未来可能的 CUDA 和 AI 工具。

第三层才是暂时用不上的高性能需求：

1. 2K 高刷游戏战未来。
2. 大型 3D CAD。
3. 本地 AI 图像生成。
4. 重度 CUDA。
5. 复杂虚拟化和多容器工作流。

这样拆完之后，问题就很清楚了：

**我需要 Windows，但目前并不需要一台昂贵的高性能主机。**

这台机器更合理的定位应该是：**Windows 兼容生态 + 可扩展的性能节点。**

## 看看 mini？

需求降级以后，我又想到了 mini 主机。

Windows 微主机这些年的地位实在不好过。就像我标题所说的一样，通常来说，对于大多数的 PC 用户，微主机都是一个处于折中方案的伪需求。既不够性能，又不够便携，拓展性没有 ITX 那么强，性价比又远低于其他品类。

但是如我开头说过的一样，Mini 型的机型一直以来对我都有莫名的吸引力。无论是最近因为 AI 爆火的 Mac mini，还是曾经悄然落幕的 iPhone mini 系列，小巧、精致、性能强大的设备通常会成为我的首选。更重要的是，从第一性原理出发，在不同的方案之间，微主机是不是真正足够契合我的实际需求？

一开始看的是 780M 核显平台，比如：

1. 7840HS。
2. 8845HS。
3. 8945HS。
4. 8745HS。
5. H255 / H260。

这几颗 CPU 基本上性能都差不多，可以看作是不同的套壳。

这类机器的优点是小、省电、安静、完整度高。Radeon 780M 应付办公、轻 CAD、策略游戏和部分老网游也绰绰有余。

后来又进一步看了 890M 方案，比如 HX370 和 HX470。890M 比 780M 更强，尤其是在覆盖轻游戏和图形需求时更得心应手。但问题是，高配 890M mini 主机价格并不低，尤其是 64GB 版本经常到八九千。

如果 890M 机器都要八九千，为什么不直接上独显？

## 性能怪兽 8060S

在更高一档，我看到了 8060S，也就是 AMD 去年推出的 Ryzen AI Max+ 395 这类 Strix Halo 平台。

这类机器很强，非常强。统一内存也很有吸引力。它像是一台高端 APU 小型工作站。64GB 或 128GB 统一内存、强大的核显、小体积、较强 CPU，看起来很接近理想小主机。

但仔细想后，它并不适合我的当前阶段。因为在决策时，我遇到了和 890M 同样的问题：

1. 首先主要就是贵：单单一个核显和统一内存，就敢标到动辄一两万的售价。我猜它是为那些有明确需求的人准备的，花这么多钱买个这玩意实在是没必要。
2. 它的优势在 Windows 侧图形和 AI 工作站能力。
3. 我真正的主力生产力仍然想放在 Mac。
4. 如果真要 CUDA，NVIDIA 独显生态更直接。

8060S 确实很强，无论是在掌机还是未来的移动端生态中，它都有不可替代的位置。但它会把我的 Windows 机器推成新的主力，这和我的设备结构不一致。

## 外接显卡方案进入视野

当我看到 5060、5060 Ti 的价格在 5 月遭到了明显的下挫时，我开始考虑能不能走一个微主机外接显卡的野路子？

与其花高价追 890M 或 8060S，花大价钱买一个 GPU 小废物。不如把 Windows 主机本体做成一个稳定平台，将图形性能交给外接 NVIDIA 显卡。如果一个 5060 只要 2000 元上下，加起来反而会比 890M 主机更便宜。

我开始比较几条路线：

### 方案一：8945HS + OCuLink

优点：

1. 主机本体有 780M，单机也能用。
2. OCuLink 外接显卡性能损耗较小。
3. 价格相对合理。
4. 适合后期接 RTX 5060。

缺点：

1. 要额外买 OCuLink 显卡坞、电源和线材。
2. 桌面会比较散。
3. 不同显卡坞质量不一。

### 方案二：7945HX 强 CPU 小主机

优点：

1. 16 核 32 线程，CPU 很强。
2. 接 RTX 5060 后会变成很强的小工作站底座。
3. 适合 Docker、后端服务、浏览器自动化、编译、多任务。

缺点：

1. 核显很弱，不接显卡时游戏和图形体验差。
2. 价格一样不便宜。
3. 微主机能不能压住 7945HX 是个问题。
4. 只能装半高显卡，未来选择受限。

### 方案三：零刻 GTi14 Ultra + EX Pro Dock

这个方案慢慢从各种选择当中脱颖而出。而且坦诚的讲，最开始选择微主机时，零刻 SER 系列几台 780M、890M 就曾是我的首选。

它的逻辑是：

1. GTi14 本体已经是完整 Windows 主机。
2. 32GB 内存和 1TB SSD 到位，整体价格清晰。
3. EX Pro Dock 可以接普通桌面显卡，不用找低矮显卡。
4. 比普通 OCuLink 裸坞更整洁，并且性能损耗更低。

这套方案的 CPU 大概和 780M、890M 差不多，稍微略好一点，GPU 可能还不如刚刚说的那两个芯片，但它是完整度很高的方案。

还有一个打动我的点是，它 32GB 内存加上 1TB SSD 的价格，只比准系统高了 1800 多块钱，算上拓展坞整机控制在 5600 元左右。这在现在的装机市场上，能帮我省下了最大的一项支出。

## 为什么没有选 795S7

铭凡 795S7 其实很有吸引力——2999 元的 7945HX 准系统，CPU 很强，内置 PCIe 插槽，未来可以装独显。从性能上看，这个方案很漂亮。

但它的问题在于：

1. 需要自己补内存和硬盘。官方的 32GB 加 1TB 的配置到 6000 多了。
2. 未来显卡需要 Low Profile 低矮版，普通 RTX 5060 不一定能装进去。
3. 低矮版显卡的型号、价格和供货不确定，省下来的钱，在显卡溢价上又补回去了。
4. 它更像小型台式机，少了一些 mini 主机的省心感。

如果我确定要走小型独显主机路线，795S7 可能很合适。  
但我想要的是一台先能独立使用、后续可以平滑接普通桌面显卡的 Windows 节点。

所以它最终输给了 GTi14 + EX Pro。

## 最终选择

最后我选择了零刻 GTi14。

这个整体组合不是性价比最高的，不是性能最强的，不是各方面都最极致的。同价位也一定有比它性价比更高、性能更强的选择。但我认为它极强的可扩展性，和足够有竞争力的价格，还有刚刚好的性能，最适合我现在的需求。

我对于一台 Windows 主机的所有需求，目前它都可以满足：

1. Windows 办公。
2. 政务、税务、银行、公司事务。
3. 轻度 CAD。
4. 浏览器自动化。
5. 插件测试。
6. 后端测试。
7. 策略游戏和轻度 Windows 游戏。
8. 后续接 RTX 5060 后，承担 CUDA、游戏、AI 图像、重一点的 CAD。

它独立工作时就能基本满足日常需求，等真正需要图形性能时，再接 EX Pro 和对应需求的显卡就行。这能让我避免一开始就把预算砸在高端独显或 8060S 上，也避免买一台几千块的普通 mini 主机后很快遇到性能扩展上限。

## 结语

这次选机是我定义不同设备之间角色分配的好机会，也是一次对于自身需求的重新审视：

第一，Mac 仍然是我的主力生产力平台。

第二，Windows 机器的价值在于补足 Mac 无法覆盖的场景。

第三，如果能通过扩展坞获得桌面显卡能力，那么可以在适度让步核显能力的前提下，寻找更好的 CPU。

第四，890M 和 8060S 很有吸引力，但价格一旦接近独显方案，逻辑就需要重新审视。

第五，设备选择要看长期分工、需求与预算、升级路径等等。

所以最终我的设备结构会变成：

**Mac mini：主力 Coding、创作、写作和生产力中心。**

**GTi14：Windows 兼容节点、轻 CAD、自动化、游戏和未来 RTX 扩展节点。**

**MacBook Air：移动办公和远程控制终端。**

**树莓派 5：边缘主机、监控和开发板。**

这个结构比单纯堆一台高性能 Windows 主机更清楚，也更符合我的真实工作流。

之后我会简单聊聊，在我目前的工作流当中，他们具体承担了哪些职责，不同的设备之间又是如何高效地相互协同。
